# PatchGuard Examples

These examples are controlled local PR-style scenarios. They let you demo PatchGuard without depending on a random public GitHub pull request.

Run from the project root after installing the backend and building the Docker sandbox:

```bash
python -m pip install -e ".[dev]"
docker build -t patchguard-python-sandbox:latest -f sandbox/python/Dockerfile sandbox/python
```

To avoid OpenAI API charges while reproducing the checked-in reports, unset `OPENAI_API_KEY` for the command:

```bash
env -u OPENAI_API_KEY python -m patchguard.cli analyze-demo examples/demo_parser_bug --out examples/sample_reports/demo_parser_bug.json
env -u OPENAI_API_KEY python -m patchguard.cli analyze-demo examples/demo_security_bug --out examples/sample_reports/demo_security_bug.json
env -u OPENAI_API_KEY python -m patchguard.cli analyze-demo examples/demo_no_tests_changed --out examples/sample_reports/demo_no_tests_changed.json
```

## Demo Scenarios

- `demo_parser_bug`: a parser regression where empty input changes from `[]` to `[""]`; existing tests pass.
- `demo_security_bug`: a rule evaluator changes to unsafe `eval`; Bandit reports the issue.
- `demo_no_tests_changed`: source behavior changes with no tests present.

## Sample Reports

The reports in `sample_reports/` were generated by the real `analyze-demo` CLI command, not handwritten. They are intentionally generated with `OPENAI_API_KEY` unset, so LLM test generation is recorded as skipped.

Example output for the security demo:

```text
Status: complete
Existing tests: passed (pytest passed)
Static scans: ruff check=passed, bandit security scan=failed
Security findings: 1
Changed files: 1 (+1/-3)
Risk: 50/100 (medium)
```
